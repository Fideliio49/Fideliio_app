Fix the following 5 bugs. Do not modify anything else.

Bug 1 — Keyboard pushes content up and doesn't come back :

When the user opens a form and the keyboard appears, the screen content shifts up correctly but when the keyboard dismisses, the layout does not return to its original position
Fix : wrap every screen that contains text inputs inside KeyboardAvoidingView with the correct behavior :

iOS : behavior="padding"
Android : behavior="height"


Add keyboardShouldPersistTaps="handled" on all ScrollView components
Add dismissKeyboard on tap outside any input using TouchableWithoutFeedback wrapping the screen
This must be fixed on ALL screens with inputs : campaigns form, profile edit, login, register, settings


Bug 2 — Merchant side missing theme/color picker in profile (same as customer) :

Terracotta #C85A17

Light/dark toggle : same behavior as customer side — saves to AsyncStorage under key merchant_theme, applies instantly without restart
Place this section just above the "Se déconnecter" button in the merchant profile screen


Bug 3 — Customer home screen : status bar still overlapping the header :

The time and battery icons are still showing through the header gradient on the customer home screen
Fix :

Import StatusBar from react-native
Set StatusBar to barStyle="light-content" on this screen
Wrap the entire screen (including the gradient header) inside a SafeAreaView with style={{ flex: 1, backgroundColor: '#C85A17' }} — the backgroundColor must match the TOP color of the gradient so the status bar area blends in
Do NOT use edges prop alone — also set the background color on the SafeAreaView

Same issue as Bug 3 but on the merchant dashboard

Fix : same approach as Bug 3 but with merchant top gradient color #2C3E8C as the SafeAreaView backgroundColor
Set StatusBar barStyle="light-content" on this screen

Bug 5 — Campaigns form : keyboard type is wrong for reward name field :

The "Nom de la récompense" input is triggering the numeric keypad instead of the alphabetic keyboard

Reward name : keyboardType="default"
